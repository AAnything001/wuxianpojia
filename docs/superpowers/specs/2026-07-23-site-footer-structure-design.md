# codexpojia.com 全站页脚结构设计

## 背景

官网当前页脚按页面各自维护：首页为单行版权与入口，专题页、培训页、用户须知和知识库页面使用不同的简化布局。参考 `https://www.xlsxdiffmerge.com/license` 的页脚采用品牌、产品、条款/资源、联系四类信息，并把友情链接单独放在底部主体区域，桌面端多列、移动端纵向重排。

本次目标是借鉴该信息架构，建立无限破甲自己的全站页脚，并加入对 `https://www.xlsxdiffmerge.com/` 的友情链接。页面主体、购买流程、下载渠道、授权/CDK 逻辑和现有 SEO 内容不改变。

## 目标与非目标

### 目标

- 为全部 55 个静态 HTML 页面提供一致的页脚层级和视觉组件。
- 在桌面端提供四列可扫描信息，在平板和手机端自然重排。
- 保留现有内部入口，并增加清晰的产品、资源、条款和联系路径。
- 以静态 HTML 输出友情链接，使其可抓取、可访问，并在新窗口打开。
- 使用本地资源和共享 CSS，减少各页面重复样式，避免依赖第三方运行时。

### 非目标

- 不重做首页 Hero、价格区、下载弹窗或任何正文模块。
- 不修改网盘地址、支付地址、更新元数据、签名更新文件或激活逻辑。
- 不引入客户端渲染 footer、外部脚本、广告或统计服务。
- 不把参考站的品牌文案、版权声明或视觉身份复制到无限破甲。

## 设计

### 信息架构

页脚由三层组成：

1. `footer-main`：品牌区、产品区、资源与条款区、联系区四列。
2. `footer-partners`：独立的“友情链接”横条，包含 XlsxDiffMerge 官方标识、站名、域名和外链箭头。
3. `footer-bottom`：版权与“仅供逆向学习研究，使用时应遵守当地法律法规”声明。

统一使用根路径内部链接，避免知识库页面的相对路径漂移：

- `/`
- `/codex-pojia`
- `/培训文案`
- `/knowledge/`
- `/user-notice`
- `/#a-updates`

友情链接使用 `https://www.xlsxdiffmerge.com/`，设置 `target="_blank"` 和 `rel="noopener noreferrer"`。链接文案为 `XlsxDiffMerge`，辅助域名文案为 `xlsxdiffmerge.com`，不改变用户提供的目标 URL。

### 视觉与响应式

- 页脚采用深色中性背景，与首页现有浅色主体形成结束区域；品牌强调色沿用无限破甲的红/紫渐变，仅用于 Logo、链接 hover 和箭头。
- 桌面端（`min-width: 900px`）为品牌列加三列链接信息。
- 平板端（`601px-899px`）使用两列网格，品牌区占满第一行。
- 移动端（`max-width: 600px`）变为单列，友情链接横条内部保持图标、名称、域名和箭头不换成溢出布局。
- 所有外链拥有可见 `:focus-visible` 状态；触控目标保持至少 40px 高度。
- 友情链接是带边框的横向信息条，不制作嵌套卡片，不增加悬浮遮挡。

### 实现边界

- 新增 `assets/site-footer.css` 作为共享页脚样式。
- 每个静态 HTML 页面保留页脚 HTML（不通过 JavaScript 注入），统一替换现有 footer 标记并引用共享样式。
- 将 XlsxDiffMerge 官方 Logo 保存为本地 `assets/partners/xlsxdiffmerge-logo.png`，避免生产页依赖对方资源可用性；图片提供明确 alt 文本。
- 机械替换通过临时脚本完成后删除脚本，仓库只保留静态页面、样式和图片资源。

## 验证策略

- 统计所有 HTML 页面的页脚结构，确认每页都有 `footer-main`、`footer-partners` 和 `footer-bottom`。
- 解析所有内部链接，确认根路径链接在首页、专题页、知识库文章和协议页均不产生错误相对路径。
- 检查友情链接 URL、`target`、`rel`、Logo alt 文本和可访问名称。
- 执行 `git diff --check`，并验证现有下载、购买、更新元数据和签名文件未被改动。
- 在 Chromium 1440px 和 390px 视口检查页脚截图，确认无横向溢出、文字截断、链接重叠或焦点不可见。
- 对首页、`/codex-pojia`、`/knowledge/`、一篇知识库文章和 `/user-notice` 做线上 HTTP 验证；部署后再次确认友情链接可达。

## 回滚

页脚改动限定在 HTML footer、`assets/site-footer.css` 和友情链接 Logo 资源。若出现问题，可回滚该提交，不影响下载目录、授权文件、正文和 SEO 配置。
