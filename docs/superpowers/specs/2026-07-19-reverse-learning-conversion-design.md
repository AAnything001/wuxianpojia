# 逆向入门课程与转化文案改版设计

## 目标

把“培训文案”页从资料介绍页改成零基础用户也能读懂的逆向学习入门页。页面重点是让用户理解软件逆向研究在做什么、下一步如何思考，并在读完后自然了解“无限破甲”可以提供一套分析路线与辅助能力。

“一头猪也能看懂”作为内部可读性标准，落地为：少术语、短句、先说结论、每一步都给出具体动作和示例。

## 非目标

- 不制作无限破甲软件的安装、激活或客户端操作教程。
- 不保留“学习资料地图”、工具分类表、安装命令或 GitHub 来源链接。
- 不承诺自动破解、保证结果或替用户跳过分析判断。

## 页面结构

1. **Hero：先把逆向说成人话**
   - 主标题：逆向知道的一切，从看懂一个文件开始。
   - 副文案说明：逆向就是把程序拆开观察，弄清它做了什么。
   - 主 CTA：开始学习逆向。
   - 次 CTA：了解无限破甲如何辅助分析。

2. **课程目录：一条看得懂的学习路线**
   - 第 1 章：软件逆向到底是什么
   - 第 2 章：先认识你手里的文件
   - 第 3 章：学会问一个好问题
   - 第 4 章：不运行程序也能看什么
   - 第 5 章：运行后如何验证猜测
   - 第 6 章：把证据整理成结论
   - 第 7 章：用真实场景练习一次
   - 保留当前目录切换、进度、上一章/下一章和 hash 恢复。

3. **每章正文：一件事、一个例子、一个动作**
   - 先用一句话给结论。
   - 再解释一个必要术语，首次出现时用白话解释。
   - 提供“你现在可以做什么”的 1～3 个动作。
   - 提供一个安全、授权范围内的示例问题。
   - 不出现 GitHub 外链、安装命令或复杂工具清单。

4. **练习区：从一句普通话开始**
   - 提供 APK、EXE、网页请求、未知文件四类提问模板。
   - 模板结构统一为：我看到了什么 / 我想确认什么 / 请先告诉我下一步。

5. **购买引导：学到这里，下一步有人带路**
   - 解释无限破甲的定位：把“描述问题、安排分析顺序、整理证据”这条路线接到 AI 上。
   - 使用真实、可核验的价值表达：少在入口上绕路、知道下一步查什么、保留过程和证据。
   - 明确适用人群：想学逆向但不知道从哪里开始的人。
   - 明确边界：不能替代学习，也不保证所有目标都有结果。
   - CTA：先试 7 天 ¥19.9；长期使用 ¥68。
   - 不使用虚假稀缺、夸大成功率、恐吓或“保证破解”等话术。

## 文案原则

- 每段不超过 3 个短句，优先使用动词。
- 先讲“你会得到什么”，再讲概念名词。
- 用“看文件、找线索、做验证、写结论”替代堆叠工具名。
- 购买引导围绕节省摸索时间和获得清晰路线，不制造不真实压力。

## 验证标准

- 页面全文不出现 GitHub URL、`git clone`、`npx skills add` 等资料安装内容。
- 页面不出现“学习资料地图”或原工具分类表。
- 课程目录有 7 章，移动端可横向浏览，章节切换和 hash 恢复正常。
- 普通用户只看 Hero、课程目录和第一章，就能说出“逆向是在观察程序做了什么”。
- 购买 CTA 同时展示 7 天体验和永久版价格，并可回到首页购买入口。
- 浏览器控制台无错误，390px 视口无横向溢出。
