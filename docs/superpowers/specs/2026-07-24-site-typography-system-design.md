# 官网全站字体规范设计

日期：2026-07-24  
状态：已获用户设计方向批准，等待书面规格复核

## 背景

官网当前共有 55 个 HTML 页面，字体样式主要分散在各页面的内联 CSS 与 `assets/site-footer.css` 中。现状包含约 60 种字号写法，并普遍声明 `Inter`，但仓库没有加载或自托管该字体。结果是不同操作系统实际使用不同回退字体，标题字宽、字重、换行与正文密度不稳定。

本次只统一字体系统，不调整页面文案、颜色、布局结构、组件形态和既有动效。

## 目标

- 建立一套覆盖首页、专题页、培训页、用户须知和知识库的字体规范。
- 在 Windows、macOS、Android 与 iOS 上优先使用各平台质量可靠的系统中文字体。
- 消除未加载字体、过度负字距、伪粗体和移动端正文过小问题。
- 将后续字体调整集中到一个共享样式文件，避免继续复制 55 份规则。
- 不增加字体文件、第三方请求或页面加载体积。

## 非目标

- 不引入或下载 Inter、思源黑体等 Web Font。
- 不重构现有页面的完整 CSS。
- 不更改页面颜色、间距体系、卡片尺寸、布局断点或内容层级。
- 不修改任何营销文案、购买链接或功能逻辑。

## 实现方式

新增共享文件 `assets/typography.css`，并在所有页面的现有内联样式和其他样式表之后加载，使字体规范成为全站最后一层排版约束。全部页面统一使用 `/assets/typography.css`，保持生产站路由一致。

共享文件使用语义选择器覆盖现有字号，不删除页面原有 CSS。这样既能控制改动面，也便于通过移除单个 `<link>` 快速回滚。

### 页面范围

- `index.html`
- `codex-pojia.html`
- `培训文案.html`
- `user-notice.html`
- `knowledge/index.html`
- `knowledge/*.html` 全部知识库文章
- `assets/site-footer.css` 中与共享规范冲突的字体声明

## 字体栈

正文与界面统一使用系统无衬线字体：

```css
--font-sans: system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI",
  "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei",
  "Noto Sans CJK SC", "Noto Sans SC", Arial, sans-serif;
```

代码、版本号或需要等宽显示的内容使用：

```css
--font-mono: "Cascadia Code", "SFMono-Regular", Consolas,
  "Liberation Mono", monospace;
```

字体栈不依赖外部网络。Windows 优先使用 Segoe UI 与 Microsoft YaHei，macOS/iOS 优先使用系统字体与 PingFang SC，Android/Linux 可回退至 Noto Sans CJK SC。

## 字号与行高

| 角色 | 桌面端 | 移动端 | 行高 | 建议字重 |
|---|---:|---:|---:|---:|
| 首页主标题 | `clamp(34px, 4vw, 52px)` | 最小 34px | 1.12 | 750 |
| 普通页面 H1 | `clamp(36px, 4.5vw, 52px)` | 最小 36px | 1.16 | 750 |
| H2 | `clamp(28px, 3.5vw, 40px)` | 最小 28px | 1.22 | 720 |
| H3 | 20px | 18px | 1.4 | 700 |
| 大正文 / Lead | 18px | 17px | 1.75 | 400–500 |
| 正文 | 16px | 16px | 1.75 | 400 |
| 导航 / 按钮 | 14–15px | 14–15px | 1.4 | 650–700 |
| 辅助文字 | 14px | 14px | 1.65 | 400–600 |
| 标签 / Eyebrow | 12px | 12px | 1.5 | 700 |
| 极小免责声明 | 不低于 12px | 不低于 12px | 1.6 | 400 |

首页视觉标题允许保持明显的品牌层级，但不再使用当前最高 98px/112px 的历史声明。知识库以阅读为主，正文固定为 16px，文章导语为 18px，避免与营销页复用过大的标题比例。

## 字重规范

- 正文：400。
- 强调正文与导语：500–600。
- 导航、按钮与卡片标题：650–700。
- 页面大标题：720–750；浏览器不支持可变字重时自动使用邻近可用字重。
- 不再新增 800、850、900、950 等过度粗体声明。
- 既有品牌 Logo 文字可保留 700，但不依赖伪粗体形成识别度。

## 字距与换行

- 中文正文 `letter-spacing: 0`。
- 大标题最多使用 `-0.025em`，替代现有最低 `-0.065em` 的紧缩字距。
- Eyebrow 与全大写英文标签使用 `0.08em`，不超过 `0.1em`。
- 标题启用 `text-wrap: balance`，正文与列表不强制平衡换行。
- 正文阅读宽度由现有容器继续控制，本次不改变布局宽度。

## 响应式规则

- `html` 设置 `text-size-adjust: 100%`，防止移动浏览器任意放大或缩小文字。
- 720px 以下保持正文 16px，不通过媒体查询降为 14px 或 15px。
- H1、H2 使用 `clamp()` 自然缩放；H3 在移动端降至 18px。
- 导航、按钮和表单类交互文字不低于 14px。
- 免责声明和页脚正文不低于 12px，并保持至少 1.6 行高。

## 用户视觉复核调整

本地首轮预览后，首页首屏标题由最高 64px 收敛到 52px，移动端固定为 34px，避免两行承诺文案压过产品界面。免费模块区的旧局部规则同时被定向覆盖：模块说明正文为 16px，数据数字为 30px（移动端 28px），数据说明与分类标签为 13px。该调整只作用于首页对应区域，不改变知识库和其他页面层级。

## 层叠与兼容策略

- `typography.css` 在页面内联 `<style>` 和 `site-footer.css` 之后加载，避免修改大量压缩内联 CSS，并保证页脚也服从同一套字号下限。
- 选择器按页面语义控制：首页使用 `.a` 作用域，知识库使用 `.article`、`.article-list` 等现有结构，专题与条款页沿用现有语义类。
- 不使用 `!important` 作为常规方案；只有无法通过正常层叠覆盖的历史内联 `style` 才允许最小范围使用，并在测试中单独记录。
- 支持不认识 `text-wrap: balance` 的浏览器自然回退，不影响可读性。
- 不禁用字体合成，避免部分系统缺少特定字重时产生不可预测的回退。

## 验证标准

### 自动检查

- 所有 55 个生产 HTML 页面均引用 `/assets/typography.css`，且每页只引用一次。
- 共享文件包含字体栈、字号令牌、正文基准和移动端规则。
- 不出现远程字体 URL、`@font-face` 或新增第三方请求。
- 现有页脚、发布元数据和首页新版本标签测试继续通过。
- 新增字体规范测试，校验共享文件存在、全页面接入和关键字号下限。

### 浏览器检查

至少检查以下页面：

- 首页 `/`
- 专题页 `/codex-pojia`
- 培训页 `/培训文案`
- 用户须知 `/user-notice`
- 知识库首页 `/knowledge/`
- 一篇知识库文章

桌面视口使用 1440×900，移动视口使用 390×844。检查标题换行、按钮溢出、导航高度、卡片标题、正文行距、页脚密度以及横向滚动。所有页面必须保持无横向溢出，正文计算字号不得低于 16px，说明文字不得低于 12px。

## 分步落地

1. 建立共享字体令牌与基础语义规则，先接入首页并验证桌面/移动端。
2. 接入根目录专题、培训和用户须知页面，验证营销页与长文页差异。
3. 批量接入知识库页面，使用自动检查确认 55 个页面覆盖完整。
4. 运行全量测试和浏览器截图检查，修正局部换行或历史高优先级声明。

每一步均保持页面可运行，并能通过移除该步新增的样式引用独立回滚。
