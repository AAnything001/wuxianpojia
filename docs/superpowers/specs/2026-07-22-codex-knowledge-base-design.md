# Codex 破甲知识库页脚方案设计

## 目标

在不破坏首页和 `codex-pojia` 专题页转化区的前提下，新增一个低调可见的 `Codex 破甲知识库`，用于承接长尾搜索词并持续累积可索引页面。

本次先落地 50 篇静态文章，围绕以下主轴展开：

- `codexpojia`
- `codex破甲`
- `codex逆向`
- `ai逆向`
- `codex逆向`

## 结构

新增两个层级：

1. 知识库首页：汇总 5 个分类、50 篇文章、少量精选推荐。
2. 文章页：每篇文章使用独立 HTML 文件、独立 `title`、`description`、`canonical` 和基础结构化数据。

推荐路径：

- `/knowledge/`：知识库首页
- `/knowledge/<slug>`：文章页

## 入口位置

首页和 `codex-pojia` 页脚新增低调文字入口：

- `Codex 破甲知识库`

知识库首页底部保留返回首页和专题页的弱化链接，不增加按钮、动画或首屏干扰。

## 内容分组

50 篇文章分为 5 组，每组 10 篇：

1. 什么是 `codex破甲`
2. `codex逆向` 基础配置
3. `ai逆向` 提问方法
4. `codex逆向` 实战流程
5. 常见问题与边界说明

每篇文章只解决一个问题，避免重复堆词或隐藏文本。

## SEO 规则

- 所有页面保持静态可抓取。
- 文章标题和首段自然包含目标关键词，不做隐藏关键词。
- 知识库首页只展示少量精选链接，其余通过分类页和站内链接串联。
- 更新 `sitemap.xml`，加入知识库首页和全部文章页。
- `lastmod` 使用本次部署日期。

## 验证

- 首页和专题页的可见转化内容不被改动。
- 新增页面可通过静态访问返回 `200`。
- `sitemap.xml` 可解析且包含全部新增 canonical URL。
- 线上页面能看到页脚入口、知识库首页和 50 篇文章页。
