# 用户使用须知协议页面设计

## 目标

为无限破甲新增一个可独立访问、可分享的 `用户使用须知协议` 页面，并在现有页脚链接组中增加入口。协议只说明授权范围、合法使用和免责声明，不扩展到退款、隐私、争议解决或其他条款。

本次改动不得影响首页购买入口、下载流程、CDK、激活校验或现有转化内容。

## 页面与入口

- 新增静态页面 `user-notice.html`。
- 页面 canonical URL 为 `https://codexpojia.com/user-notice`。
- 首页页脚链接组改为：`Codex 破甲介绍 · Codex 破甲知识库 · 用户使用须知协议`。
- `codex-pojia.html` 页脚同步增加 `用户使用须知协议`，让专题页用户也能直接访问。
- 协议页页脚提供返回首页、Codex 破甲介绍和知识库的普通文字链接。
- `sitemap.xml` 增加协议页 canonical URL，`lastmod` 使用 `2026-07-23`。

页面标题为 `用户使用须知协议`，开头说明：下载、购买、激活或继续使用无限破甲相关软件与服务，即表示用户已阅读并同意本协议。

## 视觉设计

- 沿用 `codex-pojia.html` 的品牌、字体、颜色和页脚风格。
- 页面采用单栏阅读布局，正文最大宽度约 `820px`，便于桌面端和移动端阅读。
- 三个条款使用清晰编号和轻量分隔线，不使用营销卡片、弹窗、动画或购买按钮。
- 页脚入口保持现有弱化文字链接样式，不干扰主要转化区。

## 技术实现

- 使用独立静态 HTML，不增加 JavaScript 或外部依赖。
- 为协议页添加准确的 `title`、`description`、canonical、Open Graph 和基础 `WebPage` JSON-LD。
- 所有站内链接使用与现有 Cloudflare Pages 无后缀访问方式一致的相对路径。
- 只修改 `index.html`、`codex-pojia.html`、`sitemap.xml`，并新增 `user-notice.html`。

## 验证标准

- 协议页只出现三个一级条款，且文案为无限破甲自己的版本。
- 首页和 Codex 破甲专题页均能从页脚打开协议页。
- 协议页的返回链接全部有效。
- HTML 元数据与 JSON-LD 可解析，canonical 指向无后缀 URL。
- `sitemap.xml` 可解析并包含 `https://codexpojia.com/user-notice`。
- 桌面端与移动端无文字溢出或遮挡。
- `git diff --check` 通过，现有购买、下载、CDK 和激活逻辑无差异。
