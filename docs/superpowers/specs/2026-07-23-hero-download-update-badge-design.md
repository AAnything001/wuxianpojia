# Hero 下载按钮新版本标签设计

## 目标

在官网首页 Hero 的首个“立即下载”按钮内部右侧增加一个小型“新版本”标签，让访客在主要下载入口处直接感知 Windows v1.30.1 已发布。

## 范围

- 只修改首页 Hero 中带有 `data-download-placeholder` 的首个下载按钮。
- 不修改导航栏下载入口、页面下方 Windows/macOS 下载区或最终 CTA。
- 不修改下载弹窗、网盘地址、购买地址、更新日志、CDK/激活逻辑和发布元数据。
- 本轮只提供本地预览，不推送、不部署。

## 视觉设计

- 在现有按钮内部最右侧放置胶囊标签，显示“新版本”。
- 标签使用品牌红色背景、白色文字和轻微红色阴影，与深色下载按钮形成清晰对比。
- 按钮主体文字及 Windows/macOS 图标继续保持视觉居中，标签使用绝对定位，不参与内容宽度计算。
- 桌面端标签距右侧 14px；移动端距右侧 12px，并使用更紧凑字号。
- Hero 下载按钮增加足够的右侧内边距，避免按钮内容与标签重叠。

## 动效

- 胶囊内部使用低透明度白色高光，从左向右斜向扫过。
- 每轮动画时长为 2.8 秒，前 60% 保持静止，后 40% 完成一次扫光，避免持续闪烁。
- 标签本体不缩放、不位移、不改变文字透明度，下载按钮布局保持静止。
- 高光使用 `pointer-events: none`，不产生独立交互区域。
- 在 `prefers-reduced-motion: reduce` 下停用扫光动画，保留静态红底白字样式。

## 交互与可访问性

- 标签使用非交互的 `span`，不创建嵌套按钮或独立点击区域。
- 标签设置 `aria-label="Windows v1.30.1 已发布"`，原下载按钮仍是唯一交互目标。
- 点击按钮任意位置，包括标签区域，继续打开现有下载弹窗。

## 验证

- 静态测试确认 Hero 下载按钮内恰好存在一个标签，文案和无障碍名称正确。
- 验证下载按钮仍保留 `data-download-placeholder`，现有弹窗监听逻辑不变。
- 使用 Chromium 在 1294x829、390x844 视口截图，确认标签不覆盖文字、平台图标或按钮边界。
- 确认扫光只发生在胶囊内部，不溢出标签圆角边界，也不改变按钮尺寸。
- 在减少动态效果媒体查询下确认动画已停用。
- 点击标签所在区域，确认下载弹窗正常打开。
- 执行 `git diff --check`，确认不包含发布文件、下载地址或付款地址变更。
